# PRIME v2 Form Inventory

Master catalog of all source proposal forms. Update this file as forms are analyzed, specified, approved, and implemented.

## How to Use This File

1. Each source file in `word/`, `excel/`, or `pdf/` gets one row below.
2. When a form is analyzed, change **Status** to `For Analysis` → `Specification Draft` → `For Approval` → `Approved` → `Implemented` → `Tested`.
3. When a specification is written, add the filename under **Spec File** in `converted-form-specs/`.
4. Fill in **Owner** and **Assigned Focal** during Phase 1 (Project Brief).

## Status Values

- **For Inventory** — file placed in folder, not yet analyzed
- **For Analysis** — under review for field mapping and conversion
- **Specification Draft** — web-form spec being written
- **For Approval** — spec ready for form owner review
- **Approved** — spec approved; ready for implementation
- **Implemented** — dynamic web form built in application
- **Tested** — form validated against source and acceptance criteria
