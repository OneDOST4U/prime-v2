# Converted Form Specifications

One approved specification file per source form, created during Phase 3 (Form Inventory and Conversion).

**Naming convention:** `FORM-XXX-<short-name>.md` (matches Form ID in [FORM-INVENTORY.md](../FORM-INVENTORY.md))

**Approval required before:** Web-form implementation for that form

Each specification must include: field list, validation rules, calculations, role permissions, comment permissions, and output requirements.
